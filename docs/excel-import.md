# Excel 导入格式设计

当前版本已经从固定 `Q1...Q20` 改为多学科、多考试、题目级得分模型。因此推荐使用“长表”格式，而不是把题目横向展开成 `Q1`、`Q2`、`Q3`。

## 推荐字段

| 字段 | 说明 | 示例 |
| --- | --- | --- |
| 学号 | 学生外部 ID | G8-001 |
| 姓名 | 学生姓名 | 赵六 |
| 年级 | 学生年级 | 初二 |
| 班级 | 学生班级 | 八年级1班 |
| 考试 | 考试批次 | 期中考试 |
| 日期 | 考试日期 | 2026-04-25 |
| 科目 | 学科名称 | 物理 |
| 题号 | 题目序号 | 1 |
| 满分 | 该题满分 | 10 |
| 得分 | 学生该题得分 | 8 |
| 考点 | 该题考点，可多个 | 运动与力 |

## 为什么使用长表

长表格式适合真实学校场景：

- 不同科目题目数量不同
- 不同年级学习科目不同
- 一次考试可以包含多科
- 后续可以直接聚合单科总分、每题得分率和考点掌握率
- 更容易接入老师上传试卷后的 AI 辅助考点标注流程

## 当前实现状态

当前项目已经具备数据库结构、API 和页面分析能力，但尚未实现完整 Excel 长表导入接口。

现有阶段五提供的是“AI/规则生成试卷解析草稿 -> 老师确认”的扩展点。后续如果继续增强，可以基于本格式实现正式 Excel 导入。
