# 智能化认知诊断 MVP - 项目说明书

## 项目目标
在"AI+教育"领域，精准评估学生的知识掌握程度（认知诊断），实现个性化推荐和智能体（AI Agent）辅导的基础。

## 业务概念
- **Q矩阵**（知识点-题目关联矩阵）：定义每一道题目考察了哪些具体的知识点。
- **X矩阵**（学生-题目作答矩阵）：记录了学生在测试中的真实表现（对/错）。
- **核心目标**：利用项目反应理论（IRT）或相关算法，结合知识图谱逻辑，通过分析Q/X矩阵，产出学生的知识掌握概率画像。

## 技术栈与约束
- **必须使用 Codex**（接入 Kimi 模型）
- **必须使用 Everything Codex (ECC)**
- 从0到1独立完成轻量级的算法服务及可视化 Demo
- 数据库：SQLite
- 算法：DINA模型（基础IRT参数估计逻辑）
- 前端：极简Web界面，展示"认知雷达图"或"知识掌握地图"

## 三大开发范式（必须体现）
1. **SDD** (Spec/Schema-Driven Development, 契约/模型驱动开发)
   - 定义数据库Schema、API接口契约及数据类型
   - 为后续代码生成提供明确的"锚点"
2. **DDD** (Design-Driven Development, 设计驱动开发)
   - 生成UI/UX Pro Max级别的设计稿和交互设计
   - 深度驱动前端组件与状态流转
3. **TDD** (Test-Driven Development, 测试驱动开发)
   - 先写测试用例，再写实现代码
   - 包含E2E测试，保障整体链路可靠性

## 开发任务清单

### 数据层
- [ ] 使用SQLite存储预设的Q矩阵（5个知识点 × 20道题）
- [ ] 使用SQLite存储X矩阵（至少10名学生的作答记录）
- [ ] 构建一个极简的知识图谱结构，展示知识点之间的前序/后继关系

### 算法层
- [ ] 实现一个基础的IRT参数估计逻辑（DINA模型）
- [ ] 结合Q/X矩阵，计算学生在各个知识点上的掌握概率

### 展现层
- [ ] 提供一个极简的Web界面
- [ ] 选择学生ID，实时展示其"认知雷达图"或"知识掌握地图"

### 隐藏需求（加分项）
- [ ] 利用大模型（LLM）根据诊断结果自动生成一段个性化的"学习建议"
